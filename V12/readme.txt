此版本开始来完成对业务的支持
通过用户注册的业务流程，实现服务端的业务处理。

注册的流程:
1:用户请求注册页面
2:在注册页面上输入注册信息并点击注册按钮
3:数据提交到服务端，服务端通过解析请求得到用户上传数据
4:根据请求路径判断是否为处理业务并处理该注册操作
5:处理完毕后响应客户端注册结果页面

通过两个版本完成注册，此版本完成两个环节:
1:用户如何将数据提交给服务端(知识点:html中表单的应用)
2:服务端如何通过解析请求获取提交的数据(GET请求提交数据时URL格式的学习)

实现:
1:在webapps/myweb/下新建一个注册页面:reg.html
  在这个页面上学习表单的应用

2:当页面上点击注册并提交表单后，发现客户端发送过来的请求中请求行部分的抽象路径内容是一个包含参数的
  地址，如:/myweb/regUser?username=fancq&password=123456&nickname=chuanqi&age=22
  因此我们服务端在解析请求时要对uri的值做进一步解析，将其中的请求部分(?左侧内容)和参数部分(?右侧内容)
  分别拆分出来，并且每个参数也要单独拆分出来以便后期处理请求时获取到。
  在HttpRequest中再定义三个属性:
  private String requestURI  用来保存uri中的请求部分
  private String queryString 用来保存uri中的参数部分
  private Map<String,String> parameters 用来保存每一组参数。key:存参数名 value:存参数值
